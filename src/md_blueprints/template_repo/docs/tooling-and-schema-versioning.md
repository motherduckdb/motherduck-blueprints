# Upgrade and Migrations

This repository was generated by `md-blueprints __MD_BLUEPRINTS_VERSION__`.

The generated files are project-owned source. The `md-blueprints` package and GitHub Action are the versioned MotherDuck contract for validation, rendering, planning, deployment, cleanup, update checks, and migrations.

## Pins

The local CLI version is pinned in `Makefile`:

```make
CLI_VERSION := __MD_BLUEPRINTS_VERSION__
```

GitHub workflows pin the current action major:

```yaml
- uses: motherduckdb/motherduck-blueprints@__MD_BLUEPRINTS_ACTION_TAG__
  with:
    command: validate
```

Dependabot opens pull requests when newer action releases are available.

## Normal Upgrade Loop

Review a tooling bump in a pull request:

```bash
make setup
make doctor
make migrate
make validate
make preview-smoke wikipedia-pageviews
```

If you use live local plan/deploy/cleanup commands, install the deploy extra:

```bash
make install-deploy
```

## Compatibility

| action / CLI | schemaVersions supported | upgrade path |
| --- | --- | --- |
| v0.x | 1 | Current pre-1.0 contract |
| v1.x | 1 | First stable customer contract |
| v2.x | 1 deprecated, removed in v3; 2 current | Run `make doctor` and `make migrate` before bumping |

Minor releases are additive and should be safe to accept after preview validation. Major releases can introduce a new `schemaVersion`; run `doctor` and `migrate` first.

## requiredCliVersion

Root manifests may declare a minimum CLI requirement:

```yaml
schemaVersion: 1
requiredCliVersion: ">=1.3"
```

Validation checks this before schema details and fails with a direct pin-bump message when the installed CLI is too old.

## Migrations

`md-blueprints migrate` is dry-run by default and writes only with `--write`.

For the current `schemaVersion: 1`, `make migrate` should report that no migration is needed.
